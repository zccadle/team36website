Brief Description of Each Component

  • Component Description: BarChartComponent
    The BarChartComponent is a React functional component designed to render a bar chart using D3.js library. It fetches data from an API endpoint, processes it, and dynamically generates the bar chart based on the received data. The component utilizes state variables to manage the bar chart data and SVG references. It employs the useEffect hook to fetch and process data on component mount and to render and update the chart based on changes in dimensions or data. The rendered bar chart includes interactive features such as hover tooltips and zoom functionality, enhancing user engagement and exploration of the data. Overall, the BarChartComponent serves as a flexible and user-friendly tool for visualizing appeal data within the IFRC Chart Gallery Project.

  • Component Description: LineChartComponent
    The LineChartComponent is a React functional component designed to render a line chart using D3.js library. It fetches data from an API endpoint, processes it to prepare the data for each region, and dynamically generates the line chart based on the received data. The component utilizes state variables to manage the line chart data and SVG references. It employs the useEffect hook to fetch and process data on component mount and to render and update the chart based on changes in dimensions or data. The rendered line chart includes interactive features such as zoom functionality and tooltips, enhancing user engagement and exploration of the data. Overall, the LineChartComponent serves as a flexible and user-friendly tool for visualizing appeal data within the IFRC Chart Gallery Project.

  • Component Description: BubbleChartComponent
    The BubbleChartComponent is a React functional component designed to render a bubble chart using data fetched from an API endpoint. The component utilizes state variables to manage the bubble chart data and SVG references. It fetches data from the API, processes it to extract relevant information, and filters the data based on predefined thresholds. The component dynamically generates the bubble chart based on the received data and renders interactive features such as tooltips and zoom functionality. The rendered bubble chart includes circles representing data points, with circle sizes proportional to the number of beneficiaries. Additionally, text labels are provided for each bubble to enhance data interpretation. Overall, the BubbleChartComponent serves as an effective tool for visualizing appeal data within the IFRC Chart Gallery Project.

  • Component Description: ChordDiagramComponent
    The ChordDiagramComponent is a React functional component designed to render a chord diagram representing the flow between sectors. The component utilizes a matrix to define the flow of resources from one sector to another. It generates a chord layout based on the provided matrix data, utilizing D3.js to render arcs for each sector and ribbons connecting sectors based on the flow of resources. The rendered diagram includes interactive features such as tooltips displaying the amount of resources transferred between sectors. Additionally, the component implements zoom functionality to allow users to interactively explore the chord diagram. Overall, the ChordDiagramComponent provides an effective visualization of resource flow between sectors within the context of the IFRC Chart Gallery Project.

  • Component Description: DoughnutChartComponent
    The DoughnutChartComponent is a React functional component designed to render a doughnut chart representing data fetched from an API. This component utilizes D3.js library for data visualization. The doughnut chart displays the distribution of data points based on their categories. It dynamically fetches data from the specified API endpoint and processes it to count occurrences of each category. The resulting data is then visualized as a doughnut chart where each category is represented as a segment. The component implements interactive features such as zoom functionality and drop shadow effect for visual enhancement. Overall, the DoughnutChartComponent provides an effective visualization of data distribution in the context of the IFRC Chart Gallery Project.

  • Component Description: StreamGraphComponent
    The StreamGraphComponent is a React functional component designed to render a stream graph representing data over time. This component utilizes D3.js library for data visualization. The stream graph displays the variation of resource allocation across different sectors over a specific period. The data is hardcoded within the component, representing resource allocation in various sectors for different months. The stream graph provides insights into the trends of resource allocation over time, allowing users to identify patterns and variations in allocation across different sectors. The component also implements interactive features such as zoom functionality for better exploration of the data. Overall, the StreamGraphComponent offers an effective visualization of resource allocation trends in the context of the IFRC Chart Gallery Project.

  • Component Description: HeatmapComponent
    The HeatmapComponent is a React functional component designed to render a heatmap visualization representing the distribution of resource allocation requests across different countries. This component utilizes the ArcGIS API for JavaScript for mapping and visualization purposes. The heatmap displays the concentration of resource allocation requests by overlaying heat gradients on a map of the world. The intensity of the heatmap corresponds to the number of resource allocation requests made by each country, providing insights into areas with higher demand for resources. The data for the heatmap is fetched from two separate APIs: one for appeal data and another for country data. The component converts the fetched data into graphics, with each country represented by a point on the map. The intensity of the heat at each point is determined by the amount of resources requested by the respective country. Additionally, the component adds labels next to each point to display the exact amount of resources requested. Overall, the HeatmapComponent offers an interactive and visually appealing visualization of resource allocation requests across countries, aiding in decision-making and resource planning for the IFRC Chart Gallery Project.

  • Component Description: ForceDirectedGraphComponent
    The ForceDirectedGraphComponent is a React functional component that renders a force-directed graph visualization representing the relationships between donors and donor groups within the International Federation of Red Cross and Red Crescent Societies (IFRC). This component fetches data from the IFRC API to obtain information about donors and their respective groups. It then processes the fetched data to create nodes and links for the force-directed graph. Each donor and donor group is represented as a node, and relationships between them are represented as links. The component utilizes the D3.js library to create and manipulate SVG elements for rendering the graph. The force-directed layout algorithm is applied to position the nodes and links based on the forces of attraction and repulsion between them, resulting in an aesthetically pleasing layout that highlights the connections between donors and groups. Additionally, the component provides interactive features such as node dragging and zooming for enhanced exploration of the graph. Overall, the ForceDirectedGraphComponent offers a visual representation of donor relationships within the IFRC, aiding in understanding the network of contributions and collaborations among donors and donor groups.

  • Component Description: HierarchicalLayoutComponent
    The HierarchicalLayoutComponent is a React functional component responsible for rendering a hierarchical layout visualization of project data from the International Federation of Red Cross and Red Crescent Societies (IFRC) API. This component fetches data from the IFRC API, organizes it hierarchically based on region and country, and then visualizes the hierarchical structure using a force-directed graph layout. Each region and country is represented as a node in the graph, with projects belonging to each country nested within their respective country nodes. The hierarchical structure is rendered using D3.js, a JavaScript library for data visualization, which creates SVG elements for nodes and links in the graph. The force-directed layout algorithm positions the nodes and links based on attractive and repulsive forces, resulting in a visually appealing representation of the hierarchical data. The component also provides interactive features such as node dragging and zooming for enhanced exploration of the hierarchical layout. Overall, the HierarchicalLayoutComponent offers a clear and intuitive visualization of project data organized by region and country within the IFRC, facilitating the understanding of project distributions and relationships.

  • Component Description: App
    The App component serves as the main entry point for the application. It is responsible for rendering the entire user interface and managing the state of different components. Here's a breakdown of its key functionalities:

    1. State Management: The component utilizes React's useState hook to manage state variables such as svgSize, graphSize, and activeComponent. These state variables store information about the dimensions of SVG elements, the size of resizable graph components, and the currently active component, respectively.

    2. Event Handling: The handleGraphResize function is responsible for updating the graphSize state when the user resizes the graph components. It listens for resize events and adjusts the size accordingly.

    3. Effect Hooks: The component utilizes React's useEffect hook to handle side effects such as updating the dimensions of SVG elements (svgSize) and graph components (graphSize) when the window is resized. It ensures that the visualizations remain responsive to changes in the viewport size.

    4. Rendering Graph Components: The renderGraphComponent function dynamically renders different types of data visualization components based on the activeComponent state variable. It switches between components such as hierarchical layout, force-directed graph, chord diagram, stream graph, doughnut chart, heat map, bar chart, line chart, bubble chart, and the dashboard itself.

    5. User Interface: The component renders a set of buttons in the button-container div, allowing users to switch between different visualization components. It also displays the title of the currently active component in the graph-title div.

    6. Footer: The component includes a Footer sub-component, which renders the footer section containing information about the chart gallery and the current year.

    Overall, the App component serves as the backbone of the application, orchestrating the rendering of various data visualizations and providing a cohesive user experience for exploring IFRC data.

APIs Used

Some graphs like the Chord Diagram, and the Stream Graph use hard-coded data due to a lack of relevant data from the IFRC GO API. The rest of the graphs use API data from the API endpoint, https://goadmin.ifrc.org/api/.
